# Workstreams

## Purpose

This document defines the parallel workstreams needed to ship the English OS MVP.

Workstreams help separate ongoing responsibilities from sequential milestones.

- product decision updates

## Content And Curation Workstream

Owns:

- initial roadmap templates
- curated resource catalog
- resource metadata
- resource collections
- writing prompts
- speaking prompts

Important principle:

Resources should be curated intentionally. V1 should not allow open user-submitted resource libraries.

## Brand And Motion Workstream

Owns:

- visual direction
- motion direction
- interface feel
- voice and tone
- future style references

Known direction:

English OS should feel calm, serious, intelligent, and motion-led. Animation should be a core part of the interface experience.

## QA And Polish Workstream

Owns:

- responsive checks
- loading states
- empty states
- error states
- browser behavior
- deployment validation
- interaction polish
- animation polish

Quality bar:

V1 should feel like a real product, not a rough prototype.
