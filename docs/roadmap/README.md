## Roadmap Principle

The roadmap should protect the project from two risks:

- building too much before the product works
- shipping something that looks like a prototype instead of a real MVP

V1 should be narrow, polished, and structurally ready for future AI and server sync.

## Progress Tracking Rule

[V1 MVP Roadmap](./v1-mvp-roadmap.md) is also the active progress tracker.

Future agents should:

- find the first unchecked phase or task
- continue work from that point
- mark completed tasks with `[x]`
- avoid marking a phase complete until completion criteria are true
