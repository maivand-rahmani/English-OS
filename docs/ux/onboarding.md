# Onboarding

## Purpose

This document defines how a first-time learner enters English OS and receives an initial setup.

The onboarding flow should create clarity fast, gather enough information to build a useful system, and lead the learner into a meaningful first dashboard state.

## Onboarding Goals

Onboarding should:

- reduce uncertainty
- create an initial learner profile
- generate a useful first roadmap
- personalize early recommendations
- make the learner feel understood

It should not feel like a long placement exam or a generic account setup form.

## Entry Experience

The user should begin from a simple start state with a strong call to action such as "Start" or equivalent.

The emotional job of the entry experience is:

- this product will organize your English learning
- you do not need to figure everything out alone

## Required Inputs

V1 onboarding should collect the minimum information needed to generate an initial system:

- current level
- main goal
- available study time
- strongest skill
- weakest skill
- preferred learning formats
- main pain point

## Level Strategy

V1 should use self-reported level rather than a heavy diagnostic system.

Reason:

- faster onboarding
- lower complexity
- easier to ship cleanly
- enough accuracy for template-based personalization in V1

Light calibration can be added later if needed.

## Main Goal Types

The onboarding should support broad practical goals, for example:

- build a strong foundation
- speak more confidently
- study more consistently
- improve daily comprehension
- improve writing and expression

The exact wording can evolve later, but the system should capture goal direction from day one.

## Profile Generation

After collecting answers, the system should produce:

- a learner profile
- an initial roadmap direction
- an initial set of recommended resources
- an expected daily study rhythm

This should feel like the system has started working on the learner's behalf.

## First Dashboard State

After onboarding, the user should land directly on Dashboard.

The first dashboard state should not be empty.

It should show:

- a short personalized summary
- the current roadmap stage
- the best next action
- the first recommended resource
- a simple daily plan

This moment should deliver the first product "click":

I understand what this system is going to do for me.

## Onboarding Principles

1. Ask only what is needed to generate value.
2. Prefer speed with enough signal over perfect precision.
3. Show output immediately after input.
4. Make the first post-onboarding screen feel alive and relevant.
