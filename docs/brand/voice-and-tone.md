# Voice And Tone

## Purpose

This document defines how English OS communicates in product copy and system messaging.

Voice and tone should support the product's role as a calm, intelligent learning control center.

## Voice Characteristics

English OS should sound:

- clear
- calm
- precise
- supportive
- confident
- personally aware

It should not sound like:

- a hype coach
- a school textbook
- a childish learning game
- a generic AI chatbot
- a corporate analytics dashboard

## Tone Rules

Product copy should:

- reduce anxiety
- make the next step obvious
- explain why something matters
- avoid overpromising
- avoid fake excitement
- stay short inside dashboard surfaces

## Guidance Language

The product should guide the learner with practical language.

Good patterns:

- Start here today.
- This supports your current roadmap stage.
- Review this before moving forward.
- Speaking has been lighter this week.
- This resource is useful because it targets your current weak area.

Avoid vague patterns:

- Keep going!
- Great job crushing it!
- Unlock your potential!
- AI has optimized your journey!

## Feedback Style

Feedback should be:

- specific
- actionable
- not overwhelming
- focused on improvement
- connected to next practice

Writing and speaking feedback should leave the learner with a clear next action.

## Error Message Style

Error messages should be calm and practical.

They should explain:

- what happened
- what the user can do next
- whether progress was saved

Do not make errors cute or overly dramatic.

## Motion Copy

Because the product is motion-led, copy should stay compact enough to work with animated UI states.

Animated dashboard and feedback surfaces should use short labels, clear verbs, and concise explanations.

## UI Label Style

Because the interface direction uses compact controls, cards, and animated panels, labels should be short.

Good label patterns:

- Today
- Review
- Next resource
- Current stage
- Start speaking
- Get feedback
- Mark complete
- Needs review

Avoid long labels inside compact controls.
