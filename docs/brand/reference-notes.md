# Reference Notes

## Purpose

This document captures visual and structural notes from style references.

References should guide direction. They should not be copied literally.

## Reference Set 1: Dashboard And Speaking Structure

The first references show an English-learning dashboard and speaking interface with a shared app shell.

## What To Borrow

- large rounded central app canvas
- soft pastel atmospheric background
- left vertical icon sidebar
- centered pill-based top navigation
- black active navigation state
- black primary action buttons
- right-side utility controls
- large content panels with soft surfaces
- dashboard layout with main content and right recommendation column
- speaking layout with main active media/work area and right contextual chat panel
- tactile cards and controls
- premium soft UI feeling

## What To Adapt For English OS

The references include course-like language and lesson framing.

English OS should adapt the structure toward:

- dashboard as control center
- roadmap as learning path
- resources as curated library
- writing as workspace
- speaking as workspace
- review and recommendations as system intelligence

Do not let the UI drift into course-platform behavior just because the reference uses course cards.

## Structure Implications

Dashboard can use:

- large main progress/intelligence panel
- right-side recommendation/resource panel
- lower progress/review/activity cards
- active top-nav state
- vertical sidebar utilities

Speaking can use:

- large primary practice area
- secondary participant/prompt list
- transcript or feedback area
- right-side context panel
- bottom action controls

Writing can use a similar workspace pattern:

- large draft area
- right-side feedback/context panel
- bottom or side action controls
- animated feedback reveal

## Motion Implications

The references imply a UI that should feel animated and tactile.

Potential motion:

- pill nav active state slides between tabs
- panels enter with stagger
- charts draw in
- progress bars animate
- active cards lift subtly
- speaking waveform moves during recording
- feedback panels unfold
- right contextual panels transition by state

## What To Avoid

- copying avatars or people-centric course vibe too literally
- making English OS feel like a live class platform
- making the product too pink-only or pastel-only
- losing dashboard seriousness
- using motion only as decoration
- overcrowding the interface with social/chat elements

## Current Interpretation

These references are strongest for:

- structure
- surface feel
- navigation shape
- motion expectation
- first-glance product polish

They support the selected [Soft Liquid OS](./style-doctrine.md) direction.

They are not final decisions for:

- exact color palette
- exact typography
- exact component styling
- final density
- final icon set
