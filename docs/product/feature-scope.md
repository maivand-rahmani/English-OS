# Feature Scope

## Purpose

This document defines what belongs in V1, what may be added only if time and quality allow, and what should explicitly stay out.

It exists to protect English OS from becoming a bloated course platform or an AI gimmick.

## Scope Principles

Every feature should be tested against these questions:

- Does it improve clarity for the learner?
- Does it improve guidance or useful practice?
- Does it strengthen the system rather than inflate the product?
- Does it fit the operating-system thesis?
- Can it be implemented cleanly in V1 without turning into a maintenance burden?

If the answer is mostly no, the feature should not be in V1.

## Must-Have Features

These are core to the V1 product:

- learner onboarding and profile setup
- dashboard as the main control center
- personal roadmap
- curated resource library
- resource recommendation context
- unified progress tracking
- review and repetition queue
- writing workspace
- speaking workspace
- saved learning history
- basic learner insights derived from system activity

## Should-Have If Quality Allows

These are valuable but should not compromise focus or execution quality:

- richer weekly summaries
- more advanced resource collections
- better cross-linking between roadmap, review, and practice
- stronger mistake pattern surfacing in writing
- better speaking session reflection
- more refined insight cards on the dashboard

## Later-Phase Features

These should be intentionally delayed until the core system is strong:

- deeper adaptive AI roadmap updates
- advanced retention analysis
- a dedicated listening workspace
- broader reading-specific tooling
- vocabulary memory subsystems beyond core review
- mobile-native experiences
- social, community, or accountability layers
- highly personalized coaching flows

## Explicit Non-Goals For V1

These should not be part of the first version:

- a giant internal lesson library
- building a full replacement for external resources
- open-ended user-submitted resource libraries
- teacher marketplace features
- social feed mechanics
- aggressive gamification
- exam-prep-first product design
- AI as the main product identity

## Scope Boundaries For Key Areas

### Resources

V1 should focus on curated resources selected by the product, not on allowing users to build their own resource graph.

### AI

V1 should use AI as support for insight and feedback, not as the system that holds the whole product together.

### Writing And Speaking

These should be real workspaces, but not fully expanded tutoring platforms yet.

### Review

Review should exist in V1 because it strengthens the operating-system thesis, even if the first version is rule-based and relatively simple.
